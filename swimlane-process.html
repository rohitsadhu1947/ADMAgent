Deep Dive: Core Process Flows

Each of the three ADM core responsibilities broken down step by step

📝 Feedback Loop - Standardized Collection & AI Analysis

From scattered Excel files to structured, actionable intelligence
①

ADM Contacts Agent

ADM reaches out to assigned dormant agent via WhatsApp or phone call. Gets a talking-points brief with agent history, last policy date, and dormancy duration.

WhatsApp Phone Call
②

Structured Feedback Capture

After conversation, ADM logs feedback via WhatsApp bot. Options: (a) Voice note - AI transcribes and categorizes, (b) Interactive buttons - select from standardized dormancy categories like "System Issues", "Commission Concerns", "Market Conditions", "Product Complexity", "Personal Reasons", "Competition", "Support Issues". Each category has sub-options for granular data.

WhatsApp Bot Voice Note AI Categorization
③

AI Analysis & Pattern Detection

AI engine processes feedback in real-time: sentiment scoring, theme extraction, trend identification across regions/products. Detects systemic issues (e.g., "30% of agents in UP report portal downtime") vs individual issues.

AI Engine Auto-Processing
④

Action Recommendation

Based on feedback category, AI generates specific action playbooks for ADM: "Agent reported commission delay - Escalate to finance with ticket #, share commission structure document, schedule follow-up in 5 days". ADM receives these as actionable next steps.

AI Playbook WhatsApp Alert
⑤

Axis Team Dashboard

Aggregated insights flow to Axis leadership: feedback heatmaps by region, top dormancy reasons trending, ADM performance metrics, systemic issues requiring organizational action. Auto-generated weekly/monthly reports.

Web Dashboard Auto-Reports

Agent Queue Management

ADM sees prioritized queue: "Red" (overdue follow-up), "Yellow" (due today), "Green" (upcoming). Each agent card shows: name, last contact date, dormancy duration, previous feedback summary, recommended action.

📝

Conversation Logging

After every agent interaction, ADM logs: outcome (connected/not answered/busy), conversation summary (voice note auto-transcribed), next action required, follow-up date. All via WhatsApp - no separate app needed.

🔔

Smart Reminders

System sends WhatsApp reminders: "Follow-up with Rajesh Kumar due today. Last conversation: He mentioned portal issues. Status: Axis IT team resolved the bug last week. Suggest informing him."

📈

Performance Tracking

ADM can check their own stats anytime: "Show my stats" on WhatsApp. Gets: agents contacted this week, activation rate, pending follow-ups, comparison with peers (anonymized).

Assignment Criteria

Geography: ADM and agent in same pin code cluster / district
Language: Matching preferred language (Hindi, English, regional)
Capacity: ADM's current active agent load vs max capacity
History: ADM's success rate with similar dormancy profiles
Priority: High-value agents (previously productive) assigned first

②

Rebalancing

Weekly auto-rebalance: if an ADM is underperforming or overloaded, agents are redistributed. ADMs leaving are auto-reassigned. New ADMs onboarded get gradual ramp-up load.
